## Recommended Architecture

```
Input Content
    ↓
[Haiku - Fast Classification] ← checks basic format/safety

- **Sonnet** handles the bulk of review work — best cost/quality ratio for multi-dimensional checks

        >>> sort_dicts(items, "price", missing_key_strategy="fill", default=0)
        [{'id': 2}, {'id': 3, 'price': 30}, {'id': 1, 'price': 50}]
    """
    # Validate missing_key_strategy
    valid_strategies = {"skip", "fill", "raise"}
    if missing_key_strategy not in valid_strategies:
        raise ValueError(
            f"missing_key_strategy must be one of {valid_strategies}, "
            f"got '{missing_key_strategy}'"
        )
    
    # Handle "raise" strategy
    if missing_key_strategy == "raise":
        if any(key not in d for d in data):
            missing_indices = [i for i, d in enumerate(data) if key not in d]
            raise KeyError(
                f"Key '{key}' not found in dictionaries at indices: {missing_indices}"
            )
        return sorted(data, key=lambda d: d[key], reverse=reverse)
    
    # Handle "skip" strategy
    if missing_key_strategy == "skip":
        filtered_data = [d for d in data if key in d]
        return sorted(filtered_data, key=lambda d: d[key], reverse=reverse)
    
    # Handle "fill" strategy
    if missing_key_strategy == "fill":
        return sorted(
            data,
            key=lambda d: d.get(key, default),
            reverse=reverse
        )

```

# TCP vs UDP: Key Differences

## Overview

Both TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) are transport layer protocols, but they take fundamentally different approaches to moving data across networks.

| Aspect | TCP | UDP |
|--------|-----|-----|
| **Connection** | Connection-oriented | Connectionless |
| **Reliability** | Guaranteed delivery, in-order | Best-effort, no guarantees |
| **Speed** | Slower (due to overhead) | Faster (minimal overhead) |
| **Ordering** | Maintains packet order | No ordering guarantee |
| **Flow Control** | Yes (adaptive) | No |
| **Error Checking** | Extensive | Basic checksum only |
| **Header Size** | 20+ bytes | 8 bytes |

## How They Work

**TCP (Transmission Control Protocol)**
- Establishes a connection first (three-way handshake)
- Breaks data into ordered segments
- Verifies each packet arrived and resends if needed
- Adjusts transmission speed based on network conditions
- Closes connection gracefully when done

**UDP (User Datagram Protocol)**
- Sends data immediately without setup
- Treats each packet independently
- No acknowledgment of receipt
- Sends at a consistent rate regardless of network conditions
- No formal connection or closing

## When to Use TCP

Use TCP when **accuracy matters more than speed**:

- **Email** (SMTP, POP3, IMAP) — every message must arrive correctly
- **Web browsing** (HTTP/HTTPS) — pages must load completely and accurately
- **File transfers** (FTP, SFTP) — data integrity is critical
- **SSH/Telnet** — interactive sessions need reliable, in-order commands
- **Banking/financial transactions** — money requires guaranteed delivery
- **Database connections** — queries and responses must be precise

## When to Use UDP

Use UDP when **speed and low latency matter more than perfection**:

- **Video/audio streaming** (YouTube, Netflix, Zoom, Skype) — losing a few frames is acceptable; delays are not
- **Online gaming** (real-time multiplayer) — responsiveness beats occasional packet loss
- **VoIP/voice calls** — users tolerate minor audio glitches but can't tolerate lag
- **Live broadcasts** — can't wait for retransmissions
- **DNS queries** — simple, one-shot lookups; client can retry if needed
- **IoT sensors** — many devices sending small amounts of non-critical data
- **Network monitoring** (SNMP) — periodic snapshots where occasional loss is fine

## Practical Example

**Streaming a movie (UDP):**
- If a single frame is lost, your player just skips it — you won't notice
- If the stream waits for retransmission, the whole video buffering and playback experience gets worse

**Downloading that same movie (TCP):**
- Every byte must arrive perfectly
- The file is useless if even one bit is corrupted
- Waiting for retransmissions is worthwhile for data integrity

## Hybrid Approaches

Modern protocols sometimes use **both**:
- **QUIC** — builds TCP-like reliability *on top* of UDP for faster web connections with better mobile performance
- **WebRTC** — uses UDP for low-latency media but TCP for fallback control channels

The choice ultimately comes down to your priorities: reliability and order (TCP) or speed and low latency (UDP).
